# 2026-04-02 스프링 클라우드 및 관측성 인프라 전략 정리 (JVM 튜닝 최적화본)

## 1. 개요
본 문서는 QuietChatter 프로젝트의 인프라 비용을 최소화하고 운영 안정성을 극대화하기 위해 수립된 최종 통합 전략을 정리합니다. 개발 생산성 저하 이슈로 인해 AOT(Spring Native) 적용을 철회하고, 대신 엄격한 JVM 메모리 튜닝과 스왑 메모리를 통해 저사양 인스턴스 환경을 극복합니다.

## 2. 통합 컨트롤 플레인 (controlplane) 전략
### 시스템 심장부 통합
- 정의: 데이터 저장소와 관리 서비스(Consul)를 하나로 합쳐 `controlplane` 노드로 운영합니다.
- 포함 서비스: PostgreSQL(DB), Redis(Cache), Redpanda(Kafka), Consul(Discovery/Config).
- 사양: t4g.small (2GB RAM) + 2GB Swap Memory.
- 이점: 인스턴스 유지 비용을 최소화하며, 데이터와 관리 기능을 중앙 집중화합니다.

## 3. 리소스 최적화 핵심 전략
### 저사양 인스턴스 안정화 (AOT 대체 전략)
- 전 노드 스왑(Swap) 적용: 모든 노드에 2GB 스왑 메모리를 설정하여 물리 메모리 부족에 따른 시스템 다운(OOM)을 일차적으로 방어합니다.
- 서비스 리소스 제한: 도커 컴포즈 레벨에서 각 데이터 컨테이너의 메모리 사용량을 엄격히 제한(Redpanda 768MB, Postgres 512MB 등)합니다.

### 스프링 부트 애플리케이션 JVM 튜닝 가이드라인
`t4g.micro` (1GB) 환경에서 자바 앱을 안정적으로 구동하기 위해 배포 시 다음 사항을 필수로 적용해야 합니다.
1. 힙 메모리 고정: `-Xms256m -Xmx256m` (메모리 재할당 오버헤드 방지 및 호스트 메모리 보호)
2. 메타스페이스 제한: `-XX:MaxMetaspaceSize=128m` (무한 증식 방지)
3. 스레드 스택 축소: `-Xss256k` (기본 1MB에서 축소하여 메모리 절약)
4. 경량 GC 사용: `-XX:+UseSerialGC` (2코어 이하 환경에서 CPU/메모리 오버헤드가 가장 적음)
5. 도커 제한: 컨테이너 실행 시 `memory: 400M` 등 하드 리미트를 설정하여 앱이 호스트 전체를 마비시키지 않도록 격리.

## 4. 보안 및 접속 전략
- 무키(Keyless) 접속: 모든 노드에서 22번 포트(SSH)를 닫고 AWS SSM Session Manager를 통해 보안 터널로 접속합니다.
- 계층적 방어: NAT/Ingress(외부) -> API Gateway(진입) -> controlplane(내부) 순의 엄격한 보안 그룹 규칙을 적용합니다.

## 5. 최종 노드 구성 및 사양

| 노드 그룹 | 사양 | 주요 서비스 | 스왑 메모리 | 생성 순서 |
| :--- | :--- | :--- | :--- | :--- |
| **NAT / Ingress** | t4g.nano (0.5GB) | NAT, NGINX | 2GB | 1순위 |
| **controlplane** | t4g.small (2.0GB) | DB, Redis, Kafka, Consul | 2GB | 2순위 (NAT 의존) |
| **Application** | t4g.micro (1.0GB) | API Gateway, 마이크로서비스(JVM 튜닝 적용) | 2GB | 3순위 (NAT 의존) |

## 6. 최종 예상 인프라 비용 (서울 리전 기준)
- EC2 인스턴스 합계: 월 약 $26.60
- EBS 볼륨 및 기타 비용: 월 약 $5.00
- **최종 합계: 월 약 $31.60 (한화 약 4만 5천 원 수준)**
- 성과: AOT 없이도 JVM 튜닝과 스왑 메모리 의존 전략을 통해 t4g.micro 사양을 유지하여, 초기 권장안 대비 약 58%의 비용 절감을 달성합니다.

## 7. 향후 과제
1. 배포: `terraform apply`를 통한 실제 인프라 프로비저닝.
2. 검증: 각 노드의 스왑 메모리 활성화 여부 확인 및 Consul 구동 테스트.
3. 애플리케이션 배포 설정: 향후 마이크로서비스 도커 컨테이너 실행 시 가이드라인에 명시된 JVM 튜닝 옵션 필수 적용.
